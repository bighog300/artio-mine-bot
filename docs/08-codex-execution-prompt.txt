OBJECTIVE

Implement an AI-optional crawler mode so the ingestion system can run without OpenAI when a usable runtime map already exists.

The intended behavior is:
- if a source has an existing runtime map or applied preset, skip AI site mapping
- if deterministic classification/extraction rules are sufficient, run without OpenAI
- if no usable runtime map exists, use AI only when allowed and available
- saved presets can be applied to a source and used operationally by the crawler/extractor

REQUIRED OUTCOMES

1. OpenAI becomes optional in deterministic mode
2. `run_full_pipeline()` skips AI mapping when runtime map exists
3. presets can be applied to a source as runtime config
4. crawl/extract can operate deterministically without OpenAI
5. operator can see whether the job is deterministic or AI-assisted

BACKEND REQUIREMENTS

Implement at minimum:

- optional OpenAI config/runtime behavior
- `PipelineRunner` support for `ai_client=None`
- runtime map detection/loading helpers
- source preset apply flow
- source runtime-map retrieval/use
- deterministic-first classification/extraction
- clear skip behavior when deterministic rules do not match and AI is unavailable

Likely files include:
- app/config.py

- source mapper / preset service code
- app/db/models.py

CONSTRAINTS

- avoid unrelated refactors

- do not widen scope beyond the supplied documents
